### Critical finding (배포 차단 트리거)

**[치명버그] `index.html:38` — XSS via innerHTML**

`msg.innerHTML = 'OK: ' + email` 패턴. 공격자가 `type=email` 우회 후 `a@b<img src=x onerror=...>` 입력 시 임의 스크립트 실행.

**fix_hint**: `msg.innerHTML` → `msg.textContent` 또는 DOMPurify sanitize.

### Warning findings (배포 차단은 아님, 수정 권고)

**[무결성] `index.html:33` — 이메일 regex 느슨**

`/@/` 만 검사. `a@b`, `@`, `foo@`, `@bar`, ` @ ` 등 모두 통과.

**fix_hint**: `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` 또는 `input.checkValidity()`.

**[무결성] `index.html:18` — 비밀번호 길이 JS 측 미검사**

HTML5 `minlength="8"` 만 존재. DevTools 로 속성 제거 또는 `form.submit()` 직접 호출 시 우회.

**fix_hint**: submit 핸들러에 `if (pwd.length < 8) { showError('비밀번호 8자 이상'); return; }` 추가.

### Info findings

**[유지보수성] `index.html:32` — `pwd` dead read**

`const pwd = ...` 후 사용 없음. 비밀번호 검증 부재를 시각적으로 가린다.

**[컨벤션] `index.html:38` — 의도된 취약 코드 격리 표식 필요**

`tests/` 하위라도 의도적으로 취약한 코드는 상단 배너(`<!-- INTENTIONALLY VULNERABLE -->`) 또는 디렉터리명 변경(`tests/phase5-e2e-signup-vulnerable/`)로 식별성 부여 권고.
