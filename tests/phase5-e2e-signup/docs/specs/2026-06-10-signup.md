# Signup Feature — Design Spec

**작성일**: 2026-06-10
**목적**: agent-infra Phase 5 통합 E2E 검증용 가상 feature. 9종 산출물(spec, PRD, plan, tasks, 구현, QA 리포트, bug 리포트, review, deploy precheck)이 모두 생성되는지 확인하기 위한 최소 타겟.

이 spec 은 실제 프로덕션 feature 가 아니라 파이프라인 검증용이다. 의도된 버그 3개가 코드에 심어져 있으며, 이 버그들은 QA → integrity-review 가 발견하는지 확인하는 테스트 입력이다.

## 1. 범위

- 단일 HTML+JS 회원가입 폼 1개.
- 백엔드 없음. 입력 → JS 검증 → 화면 메시지로 종료.
- 빌드 단계 없음, 프레임워크 없음.
- 파일 1개: `index.html` (embedded `<style>`, `<script>`).
- 서빙: `python3 -m http.server 8000` 또는 file:// 직접.

## 2. 사용자 흐름

1. 사용자가 페이지를 연다.
2. 이메일과 비밀번호를 입력한다.
3. "가입" 버튼을 누른다.
4. 검증 실패 시 빨간색 에러 메시지, 성공 시 초록색 "OK: <email>" 메시지가 표시된다.

## 3. Task 분할과 의도된 버그

3개 task 로 분할한다. 각 task 는 기능 1개 + 의도된 버그 1개를 추가한다.

### T-001: 이메일 형식 검증

**기능**
- `<input id="email" type="email" required>` 추가.
- 제출 시 JS 가 정규식으로 형식 검사.
- 실패 시 `#msg` 에 "이메일 형식이 올바르지 않습니다" 빨간색.

**의도된 버그 (B-001 후보)**
- 정규식이 `/@/` (단순히 `@` 포함 여부만 검사).
- `a@b` 같은 비정상 입력이 통과한다.
- QA 가 `a@b` 입력 → "OK" 메시지가 뜨는 것을 스크린샷.

### T-002: 비밀번호 길이 검증

**기능**
- `<input id="pwd" type="password" required minlength="8">` 추가.
- 제출 시 길이 ≥ 8 검사.

**의도된 버그 (B-002 후보)**
- HTML5 `minlength` 만 사용. JS submit 핸들러에서는 길이 검증을 누락.
- 사용자가 DevTools 로 `minlength` 속성을 제거하거나 JS 로 `form.submit()` 직접 호출하면 우회 가능.
- QA 가 DevTools 로 `minlength` 제거 후 짧은 비밀번호 입력 → "OK" 메시지.

### T-003: 제출 성공 메시지

**기능**
- 모든 검증 통과 시 `#msg.innerHTML = "OK: " + email` 로 메시지 표시.
- 초록색.

**의도된 버그 (B-003 후보)**
- `textContent` 대신 `innerHTML` 사용 → XSS 취약점.
- 이메일 입력에 `<img src=x onerror=alert(1)>` 류를 넣으면 스크립트 실행 (실제로는 type=email 이라 막힐 수 있으니 검증 우회 후 직접 DOM 조작 또는 `a@b<img...>` 같은 형태로 재현).
- QA 가 페이로드 입력 → alert 또는 console 에 흔적 캡처.
- integrity-review 가 코드 단계에서 `innerHTML` 사용을 지적할 것으로 기대.

## 6. 운영 결정

- **버그 fix 라운드 없음.** 3개 의도된 버그는 코드에 남긴 채로 진행한다. E2E 목적은 산출물 생성 검증이지 버그 제거가 아니다.
- **integrity-review verdict 가 reject 여도 진행한다.** reject 리포트 자체가 산출물 #7 이며, deploy-precheck (#8) 은 secret 만 검사하므로 별개로 PASS 가능하다.
- **deploy-guard 는 commit 시점에 precheck 토큰의 신선도만 검증한다.** review verdict 와 무관.
- **모든 commit 은 사용자가 직접 실행한다.** 본 spec 작성 후에도 자동 commit 하지 않는다.
